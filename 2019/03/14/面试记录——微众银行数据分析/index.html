面试记录——微众银行数据分析

其实是微众银行的外包岗位，本着积累面试经验的原则去参加了，被提前告知这个岗位会很看重SQL，用了半天恶补，早上接到通知，下去就去微众总部了。
似乎大家都很忙，干坐了半个多小时，给了两张SQL试题让我先做着又走了。

笔试

一个大题，两个小题。大题是多表查询并创建新表，题目大意是根据客户在商户扫码和购买礼品卡的时间差，统计商户推广礼品卡的绩效，提供了giftcard，scan，和store三个表。思路是以store为关键表查询，用join语句查询其他两个表，使用的MySQL。用timestampdiff函数计算时间差（因为表中时间格式是timestamp，根据情况不同可以使用其他diff函数如datediff等）。大致解答如下图：

后面面试过程中会问到思路，join的用法，inner join， left join， 和right join的区别，以前根据这道题的实际情况选择用什么join比较合理。因为我用的是inner join，实际可能存在有商户没有任何礼品卡推广的情况，也就是在giftcard, scan表中的记录为null，所以使用left join更合理些。

两个小题，和leetcode上的题目很像。其中一个是用一条了sql余韵查询所有课程分数都大于80分的学生名字。

主要考察对题目的理解，也就是所有课程大于80分，那就可以使用最小值函数查询。
select name from _ group by name having min(score)>80;

面试

写第一题花了比较多时间，导致后面两题还没来得及做面试官就回来了，开始问思路，很惭愧，两个小题都没能回答好，但是面试官还比较有耐心，也很友好的提醒了我。
SQL方面就是这些，然后就开始正式面试，无非就是自我介绍，讲解项目等，由于我是转行的，所以项目这边能讲的就不多。然后问到一点python，主要是pandas中df的一些用法，面试官自己临时想了个题，可能觉得没出好，就作罢。
最后就是问为什么离职，为什么选择转行云云，也直接说了转行其实很难的，总之人挺好的。
最后的最后考察了一下分析的能力吧，问如果公司未作什么措施而股价却突然上涨的话，我会从哪些方面去分析。我的回答是政策形势，相关行业情况等，好像不太满意，说这些不好量化，也不好统计。

总结

总体来看，还是一次比较愉快的面试，沟通比较顺利，回答不出的时候也不至于尴尬。准备时间多点的话可能情况会好一些，sql方面确实是比较薄弱的，还是要多刷题，以及最后一问应该询问正确做法的。
最后结果是没能通过，有一点点可惜吧。
